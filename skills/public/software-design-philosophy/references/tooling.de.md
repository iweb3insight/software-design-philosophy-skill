# Multi-Tool-Kompatibilität (Claude Code / OpenCode / Gemini CLI / Codex)

Ziel: konsistente Ausgaben über CLI/IDE-Agents.

## Gemeinsame Regeln
- Nur reines Markdown.
- Diagramme:
  - Klassendiagramm: PlantUML
  - Sequenzdiagramm: Mermaid
- Struktur: Überschriften + flache Listen, keine verschachtelten Listen.

## Claude Code
- Präferenz: knapp, aufgabenorientiert.
- Erst Fazit, dann Details.
- Empfehlung: Zusammenfassung + Diagramme + Checkliste.

## OpenCode
- Präferenz: Engineering-Schritte und Artefakte.
- Nächste Schritte und Deliverables nennen.
- Empfehlung: Artefaktliste.

## Gemini CLI
- Präferenz: klare, umsetzbare Schritte.
- Wenig Hintergrund, Fokus auf Schritte/Constraints.
- Empfehlung: Ziel / Schritte / Output.

## Codex
- Präferenz: ausführbar, praxisnah.
- Referenzdatei nennen.
- Format konsistent halten.
