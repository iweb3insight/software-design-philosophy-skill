# 多工具适配说明（Claude Code / OpenCode / Gemini CLI / Codex）

目标：让同一套输出在不同 CLI/IDE 代理中保持一致。

## 通用规则
- 输出必须是纯文本 Markdown，不依赖专有格式。
- 图示统一使用：
  - 类图：PlantUML
  - 时序图：Mermaid
- 结构统一：标题 + 平铺列表，不使用嵌套列表。

## Claude Code
- 偏好：简洁、任务驱动的输出。
- 要求：先给结论，再给结构化细节。
- 输出建议：先摘要，再附图与清单。

## OpenCode
- 偏好：工程化步骤与文件指引。
- 要求：输出中明确“下一步”与“产出物”。
- 输出建议：附带产出清单（如类图/时序图/UI 结构）。

## Gemini CLI
- 偏好：分段清晰、可操作的流程。
- 要求：避免过多背景解释，强调步骤与约束。
- 输出建议：每段以“目标/步骤/产出”格式呈现。

## Codex
- 偏好：可执行、可落地的技术结构。
- 要求：明确路由规则与引用的 reference 文件。
- 输出建议：标注所用 reference 名称，保持一致格式。
